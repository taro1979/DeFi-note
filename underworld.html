Meerkat Finance: BSC Rugの開幕ベル

2021年3月、BSC上で稼働直後のVaultから約3100万ドル相当が抜かれた。Rektの分析では、Vaultの実装がアップグレードされ、初期化関数と資金抜き取り関数が追加されたことが中心だった。重要なのは、これは外部攻撃というより、管理者権限とproxy upgradeがそのまま利用者資金への支配権だった点。

  • 見るべき痕跡: proxy admin、upgradeTo、owner権限、未検証実装、timelock監視。
  • 関連: Rekt Meerkat / X検索

Compounder Finance: 監査済みでも内部者リスクは消えない

Compounderは監査済みのVaultを掲げながら、監査後に悪意あるStrategyを差し込まれた。Rektは、timelockがあっても監視されていなければ防御にならず、監査は外部攻撃だけでなく内部者権限も明示すべきだと指摘している。Rug事例では「監査済み」「timelock」「burned keys」がマーケ文句として使われやすい。

  • 見るべき痕跡: Strategy追加、Controller権限、timelock queue、監査後差分。
  • 関連: Rekt Compounder / X検索

Uranium Finance: フォーク改変の1行が全資金を壊す

Uniswap v2/PancakeSwap系のAMMをフォークし、手数料や係数を変えた結果、定数積チェックの不整合が致命傷になった。フォークRugでは、UIやAPRは派手でも、コア数式の変更レビューが薄い。2026年の報道では、Uranium Financeの約5300万ドル相当の盗難に関連して起訴も報じられた。

  • 見るべき痕跡: Uniswap v2 pair差分、fee multiplier、k invariant、未監査fork。
  • 関連: Rekt Leaderboard / X検索

PancakeBunny/Belt/bEarn: 自動複利Vaultの価格参照地獄

BSCの自動複利Vaultは、PancakeSwap LPを預けて報酬を自動で複利するだけに見えた。しかし、Vault share価格、報酬ミント、AMM価格、ステーブルプールが絡むと、Flash Loanで一時的に価格を歪めて報酬を過剰発行させられる。BSCの低手数料は利用者に優しいだけでなく、攻撃の反復試行にも優しかった。

  • 見るべき痕跡: LP価格算出、報酬mint式、単一AMM価格参照、deposit/withdraw前後のshare変化。
  • 関連: Rekt / X検索

Snowdog: OHM fork、買い戻し、情報非対称

SnowdogはAvalanche上のOHM fork/ミーム通貨実験だった。8日間の蓄積期間の後、Treasury資金で買い戻しを行うとし、専用AMMとパスワード付きフロントエンドを使った。結果として、最初の数取引が極端に有利になり、内部情報・MEV対策・公平性の境界が曖昧になった。

  • 見るべき痕跡: buyback条件、新AMM、frontend gate、事前fundingされた新規アドレス。
  • 関連: Rekt Snowdog / X検索

Iron/TITAN: Bank Run型のRugリスク

Iron Financeは運営が流動性を抜いた単純なrugではなく、部分担保ステーブルのデススパイラルだった。TITAN価格が上がるほどIRON発行とLP報酬が魅力的になり、逆回転すると償還がTITAN売りを増幅する。Rugリスクとしては、詐欺ではない設計破綻も利用者にとっては同じくらい致命的になる。

  • 見るべき痕跡: collateral ratio、償還フロー、吸収トークン流動性、大口退出。
  • 関連: Iron Finance overview / X検索

BALD: 新L2の初日が一番危ない

Base初期のBALDは、チェーンの公式導線や主要インフラが整う前にミームコインへ資金が集中した例。デプロイヤーが巨額の流動性を入れ、注目を集め、流動性を抜いた。新チェーン初期は「他に買うものがない」「公式っぽい」「橋が不便」という条件が揃い、rug pullが起きやすい。

  • 見るべき痕跡: LP所有者、流動性追加/削除履歴、初期資金の出所、橋/DEX未整備。
  • 関連: Rekt BALD / X検索

Rug Pull研究: 1-day rugとserial scammer

学術研究では、Uniswap/PancakeSwap上のscam token、1-day rug、serial scammer、Solanaの大量疑似LPなどがデータセット化されている。これは個別プロトコル史を超えて、Rugを「反復可能な詐欺オペレーション」として見る段階。名前が残らない小型rugほど、検索ではなくオンチェーン特徴量で拾う必要がある。

  • 見るべき痕跡: deployerの連続発行、LP lock偽装、honeypot、初日LP抜き、wash trading。
  • 関連: Token Spammers/Rug Pulls / Serial Scammers / SolRPDS
