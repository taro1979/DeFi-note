# CLAUDE.md ジェネレーター

プロジェクトの実態を自動解析し、AI-Ready な CLAUDE.md を生成・更新する。

## ワークフロー

### Phase 1: プロジェクト解析

以下を自動収集する（ユーザーに質問せず黙って実行）:

1. **パッケージマネージャー検出**: `package.json`, `Cargo.toml`, `pyproject.toml`, `go.mod` 等
2. **技術スタック検出**: フレームワーク、言語、主要ライブラリ
3. **ディレクトリ構造**: `ls` でトップレベル + 主要ディレクトリの内容
4. **スクリプト一覧**: package.json の scripts / Makefile のターゲット等
5. **既存 CLAUDE.md**: 存在する場合は内容を読み込み
6. **テスト構成**: テストフレームワーク、テストファイルの場所
7. **CI 設定**: `.github/workflows/`, `.gitlab-ci.yml` 等
8. **リント・フォーマッター**: eslint, biome, prettier, rustfmt 等

### Phase 2: テンプレート生成

以下の構成で CLAUDE.md を生成する。**各セクションは解析結果から自動埋め込み**し、不明な部分のみ `[TODO: ...]` で残す。

```markdown
# {プロジェクト名}

{1-2文の概要}

## 技術スタック

| カテゴリ | 技術 | バージョン |
|---|---|---|
| 言語 | TypeScript | 5.x |
| フレームワーク | Next.js (App Router) | 15.x |
| ... | ... | ... |

## アーキテクチャ

{選定理由付きで記述}

### ディレクトリ構成

```text
{実際のディレクトリツリー。各ディレクトリに1行説明付き}
```

### レイヤー依存ルール

{依存方向の制約があれば記述}

## セットアップ

```bash
{パッケージマネージャーに応じたワンコマンド}
```

### 環境変数

`.env.example` を `.env` にコピーし、以下を設定:

| 変数名 | 説明 | 必須 |
|---|---|---|
| ... | ... | ... |

## 標準コマンド

```bash
{検出したスクリプトを一覧}
```

### ワンコマンド検証（推奨）

```bash
{パッケージマネージャー} validate  # 型チェック → リント → テスト
```

## コーディング規約

### パターン例

{プロジェクトの実コードから2-3個の代表的パターンを抽出}

### インポート順序

{プロジェクトの規約に合わせて}

## Critical Rules

- {解析結果とベストプラクティスから生成}

## NEVER（絶対禁止）

- [TODO: プロジェクト固有の禁止事項を追記]
- デフォルトエクスポートを使わない（理由: ツリーシェイキングとリファクタリングの安全性）
- node_modules を直接編集しない
- .env ファイルをコミットしない
- 既存テストを削除しない
- console.log を本番コードに残さない

## 権限境界

### 自由に実行してよい
- ファイル読み取り・コード分析・編集
- テスト実行・リント実行
- ドキュメント更新

### 確認してから実行
- git push、ブランチ削除
- .env の変更
- 破壊的な DB 操作
- 外部サービスへの通信

## 読み方ガイド

| やりたいこと | まず読む | 次に読む |
|---|---|---|
| 全体像を掴む | `CLAUDE.md` | {アーキテクチャドキュメント} |
| 新機能を実装する | {仕様書の場所} | {アーキテクチャ} |
| ローカル開発 | {ローカル開発ガイド} | `package.json` |
```

### Phase 3: validate スクリプト提案

`package.json` の scripts に `validate` がない場合、追加を提案する:

```json
{
  "scripts": {
    "validate": "{typecheck} && {lint} && {test}"
  }
}
```

### Phase 4: 出力と確認

1. 生成した CLAUDE.md をユーザーに提示
2. `[TODO: ...]` が残っている箇所を明示
3. 既存 CLAUDE.md がある場合は差分を表示

## $ARGUMENTS の扱い

- `new`: 既存 CLAUDE.md を無視して新規作成
- `update`: 既存 CLAUDE.md をベースに不足セクションを追加
- 引数なし: 既存があれば `update`、なければ `new`

## 生成ルール

- **「なぜ」を付ける**: 規約には理由を添える（例: 「App Router を使う（Pages Router は段階的に廃止中）」）
- **実コード優先**: 汎用サンプルではなく、プロジェクトの実コードからパターンを抽出
- **最小限**: 推測で埋めるよりは `[TODO]` で残す
- **日本語**: プロジェクトの CLAUDE.md が日本語なら日本語で生成
