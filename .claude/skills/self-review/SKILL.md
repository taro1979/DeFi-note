# Self Review

タスクの最終段階で、完了報告の直前に1回だけ自己レビューを実行する。

## トリガー

以下のタイミングで自動実行する:
- **ファイルを1つでも編集・作成・削除したタスクの完了直前**（種類を問わない）
- `/dev-impl` や `/commit-merge` の最終ステップの前
- ユーザーに「完了しました」と報告する直前
- スキル、ルール、設定、ドキュメントの変更も対象に含む

## レビュー方法の選択

### Codex CLI が使える場合（優先）

まず `which codex` で Codex CLI の存在を確認する。利用可能なら以下のコマンドでレビューを委譲する:

```bash
codex review --uncommitted
```

- Codex の出力に問題指摘があれば、自分で修正を実施する
- Codex の出力が「問題なし」相当なら、そのまま完了報告へ進む

### Codex CLI が使えない場合（フォールバック）

以下の手順で自己レビューを実行する。

#### 1. 変更の棚卸し

今回の変更を一覧化し、以下を自問する:

- 変更ファイルに漏れはないか
- テストは追加・更新したか
- ドキュメントの更新は必要か

#### 2. 品質チェック（5項目）

| チェック項目 | 自問 |
|---|---|
| ロジック | 分岐・境界値に漏れはないか |
| 整合性 | DB・型・翻訳キーに不整合はないか |
| セキュリティ | 入力検証・認可・情報漏洩のリスクはないか |
| 既存機能 | 既存の挙動を壊していないか |
| 仕様準拠 | ユーザーの要求を正確に満たしているか |

#### 3. 判定と行動

- **問題なし** → そのまま完了報告へ進む
- **問題あり** → 修正を実施してから完了報告する。修正内容を報告に含める

## 完了マーカー

self-review を実行したら、最後に以下のコマンドを Bash で実行して完了マーカーを書き込む:

```bash
mkdir -p /tmp/claude-self-review && echo "$(date -u +%Y-%m-%dT%H:%M:%SZ)" > "/tmp/claude-self-review/${CLAUDE_SESSION_ID:-unknown}.done"
```

このマーカーは Stop hook が self-review 実行済みかを検知するために使用する。

## ルール

- ループは **1回のみ**。修正後に再度レビューは行わない
- レビュー結果をユーザーに長々と報告しない。問題がなければ触れなくてよい
- 問題を見つけて修正した場合のみ、修正内容を簡潔に報告する
- **完了マーカーの書き込みを忘れないこと**
