# ワークフローパターン

## 順次ワークフロー

複雑なタスクでは、操作を明確な順次ステップに分解します。SKILL.md の冒頭付近でプロセスの概要を Claude に提供することが効果的です：

```markdown
PDF フォームの入力には以下のステップが含まれます:

1. フォームを分析する (analyze_form.py を実行)
2. フィールドマッピングを作成する (fields.json を編集)
3. マッピングを検証する (validate_fields.py を実行)
4. フォームに入力する (fill_form.py を実行)
5. 出力を確認する (verify_output.py を実行)
```

## 条件付きワークフロー

分岐ロジックを持つタスクでは、Claude を意思決定ポイントを通じてガイドします：

```markdown
1. 変更タイプを判断する:
   **新しいコンテンツを作成する場合？** → 以下の「作成ワークフロー」に従う
   **既存のコンテンツを編集する場合？** → 以下の「編集ワークフロー」に従う

2. 作成ワークフロー: [ステップ]
3. 編集ワークフロー: [ステップ]
```

## フィードバックループ

品質重視のタスクでは、バリデーター実行 → エラー修正 → 再実行のループを組み込みます：

```markdown
## ドキュメント編集プロセス

1. `word/document.xml` を編集する
2. **即座に検証**: `python ooxml/scripts/validate.py unpacked_dir/`
3. 検証失敗の場合:
   - エラーメッセージを確認
   - XML の問題を修正
   - 再度検証を実行
4. **検証パス後のみ進行**
5. 再構築: `python ooxml/scripts/pack.py unpacked_dir/ output.docx`
```

スクリプトがない場合でも、チェックリストとリファレンスドキュメントを「バリデーター」として活用できます：

```markdown
## コンテンツレビュープロセス

1. STYLE_GUIDE.md のガイドラインに従ってコンテンツを作成
2. チェックリストで確認:
   - 用語の一貫性
   - 例が標準フォーマットに従っている
   - 必須セクションがすべて存在する
3. 問題がある場合:
   - 具体的なセクション参照と共に記録
   - コンテンツを修正
   - 再度チェックリストを確認
4. すべての要件を満たしたら完了
```

## チェックリストパターン

複雑なワークフローでは、Claude が進捗を追跡できるチェックリストを提供します：

````markdown
## PDF フォーム入力ワークフロー

このチェックリストをコピーして進捗を追跡:

```
タスク進捗:
- [ ] ステップ 1: フォームを分析 (analyze_form.py を実行)
- [ ] ステップ 2: フィールドマッピングを作成 (fields.json を編集)
- [ ] ステップ 3: マッピングを検証 (validate_fields.py を実行)
- [ ] ステップ 4: フォームに入力 (fill_form.py を実行)
- [ ] ステップ 5: 出力を確認 (verify_output.py を実行)
```
````
