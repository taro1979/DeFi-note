# doc-garden: ドキュメント健全性チェック

プロジェクトのドキュメントと実装の整合性を点検し、乖離を修正する。

## 手順

### 1. データモデル記述の確認
- `docs/domain/data-model.md` のエンティティ記述が実装と大きくズレていないか確認
- `CLAUDE.md` のドメイン説明と矛盾がないか確認

### 2. ファイルパスの実在性チェック
- `docs/architecture.md` に記載されたディレクトリや主要ファイルパスが実在するか確認
- 存在しないパスがあれば更新

### 3. コマンド一致チェック
- `CLAUDE.md` のコマンド記述が `package.json` scripts と一致しているか確認
- 不一致があれば CLAUDE.md を更新

### 4. TODO 整合性チェック
- `todo.md` の未着手項目について、対応実装がすでに存在しないか簡易確認
- 実装済みなら `[x]` 更新を検討する
- 完了済み項目が実装から消えていないかも簡易確認する

### 5. 仕様書の簡易チェック
- `spec/` 配下の仕様書を一覧
- 各仕様書の Acceptance Criteria に対応するテストや実装が存在するか簡易確認

### 6. 結果レポート
- 修正した箇所を一覧で報告
- 修正がなければ「全て整合」と報告

## 注意
- 説明文を全面的に自動生成で上書きしない
- 事実情報を中心に修正する
