# E2E テスト実行

E2E テストを実行し、失敗時は原因を特定して修正を試みる。

## ステップ 1: サーバー起動確認

- 開発サーバーが起動済みか確認する
- 自動起動設定があるなら、それを利用する

## ステップ 2: E2E テスト実行

```bash
pnpm test:e2e
```

## ステップ 3: 結果判定

### 全テストパス時
- テスト数と所要時間を報告して完了

### 失敗時
1. 失敗したテストのエラーメッセージを分析
2. 原因を切り分ける
   - セレクタ不一致
   - タイムアウトや待機不足
   - API 契約変更
   - fixture や seed データ不整合
   - 認証フロー変更
3. 修正を実施
4. 再度 `pnpm test:e2e` を実行
5. 3回失敗したら停止して報告

## ステップ 4: 既存テストへの影響確認

E2E テストの修正がある場合は、既存テストや型チェックへの影響も確認する。

```bash
pnpm test
pnpm check
```

## 注意事項
- テストを弱めて通すだけの修正はしない
- 新しい E2E テストを追加する場合は既存構成に合わせる
