# 権限エラー時の自律回避ルール

## 原則

権限エラー（Permission denied）が発生した場合、**ユーザーに確認を求めて作業を中断しない**。
代替手段を自律的に探し、タスクを完遂する。

## 回避戦略（優先度順）

### 1. 専用ツールへの切り替え
- Bash(grep ...) → Grep ツール
- Bash(cat ...)  → Read ツール
- Bash(find ...) → Glob ツール
- Bash(sed ...)  → Edit ツール
- Bash(echo > )  → Write ツール

### 2. 許可済みコマンドへの言い換え
- 許可済みの `python3 -c "..."` でワンライナー実行
- `node -e "..."` で JavaScript 処理
- 許可済みコマンドのパイプで組み合わせる

### 3. サブエージェントへの委任
直接実行が拒否される場合、Task ツールで適切なサブエージェントに委任する。

### 4. スクリプトファイル経由の実行
Write → scripts/temp_task.py → Bash(python3 scripts/temp_task.py)

## フィードバック（必須）
回避策でタスク完了後:
> 注: `{拒否されたコマンド}` が権限外のため、`{使用した代替手段}` で実行しました。

## 禁止事項
- 権限エラーだけを理由にタスクを放棄しない
- 設定ファイルを直接書き換えない
- セキュリティ制約の回避は行わない
