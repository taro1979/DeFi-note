# Self-Review 自動実行ルール

## 必須ルール

**ファイルを1つでも編集・作成・削除したら、完了報告の直前に必ず `/self-review` スキルを1回実行すること。**

ファイルの種類は問わない。コード、ドキュメント、スキル、ルール、設定ファイル、翻訳ファイルなど、すべてが対象。

### 対象タイミング

- `/dev-impl` のフェーズ3（検証）の後、フェーズ4（完了報告）の前
- `/commit-merge` のステップ2（検証）の後、ステップ3（コミット）の前
- その他、**ファイル変更を伴うすべてのタスク**の完了報告の直前

### 発火しなくてよいケース

- ファイルを一切変更していない（調査のみ、会話のみ）

### 実行方法

Skill ツールで `self-review` を呼び出す。
