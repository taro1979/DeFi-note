# 待機検知と自動進行

## 入力パターン → 行動マッピング

| ユーザー入力 | 解釈 | 行動 |
|---|---|---|
| 「ok」「うん」「はい」「了解」「lgtm」 | 承認 → 次へ | 最優先タスクを選択・即着手 |
| 「next」「続けて」「次」「go」「進めて」 | 続行要求 | 直前作業の続き or 次タスクに着手 |
| 「?」「どう？」「状況は？」 | 状況確認 | 進捗を簡潔報告 |
| 曖昧な1行指示 | 意図推定 | 仮説を宣言して着手 |

## Smart Suppression（丸投げ質問の禁止）
「何をしますか？」は禁止。「XXXを進めます。YYYも候補ですが、まずXXXからでよいですか？」が正しい。

## 自動で進めてよい操作
- ファイル読み取り・コード分析・コード編集・テスト実行・ドキュメント更新

## 必ず確認してから進める操作
- git push、ファイル・ブランチの削除、本番環境への操作、.env の変更
